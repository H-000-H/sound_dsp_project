2. 为什么选 ESP32-S3 而不是 C3 / C6

需求C3 (RISC-V 单核)C6 (RISC-V 单核)S3 (LX7 双核)
PSRAM（刚需） ❌ 不支持 ❌ 不支持 ✅ Octal SPI 8MB
Lottie 160KB 缓冲 ❌ 内部 SRAM 仅 400KB ❌ 内部 SRAM 仅 512KB ✅ PSRAM 分配，不占内部 RAM
LVGL 9.x + 240×240 ⚠️ 勉强（单核） ⚠️ 勉强（单核） ✅ 双核专用渲染
音频解码 + EQ 实时性 ❌ 无法同时 UI+音频 ❌ 无 PSRAM ✅ Core 0: UI / Core 1: 音频
屏幕刷新带宽 ❌ Quad SPI ❌ Quad SPI ✅ Octal SPI
USB-OTG ❌ ❌ ✅
S3 不可替代的原因：Lottie 渲染需要 160KB PSRAM 缓冲区，C3/C6 均无 PSRAM 控制器。双核架构允许 LVGL 渲染（Core 0）和音频解码（Core 1）分离，这是单核 RISC-V 无法做到的。

5. DeviceTree 硬件描述

JSON 格式的 mini DeviceTree（components/board/board.dts.json）作为唯一硬件配置入口：

{
    "soc": "esp32s3",
    "devices": [
        { "name": "i2s0", "compatible": "esp32,i2s-bus",
          "properties": { "ws_pin": 4, "bclk_pin": 5, "dout_pin": 6 } },
        { "name": "display", "compatible": "sitronix,st7789",
          "depends_on": "spi0",
          "properties": { "width": 240, "height": 240,
            "dc": 3, "reset": 4, "backlight": 15 } }
    ]
}

工作流程：

board.dts.json ──解析──▶ device_list
         board_register_all_drivers() ◀─── DRIVER_REGISTER 宏
            board_driver_probe_all() ──▶ driver->probe(dev)
                              设备就绪 (DEVICE_STATUS_PROBED)

更换硬件只需改 board.dts.json 中的引脚号，无需改 C 代码。

6. DSP / EQ

多段均衡器，Biquad 级联，Q31 定点实现：

        b₀ + b₁·z⁻¹ + b₂·z⁻²
H(z) = ──────────────────────
        a₀ + a₁·z⁻¹ + a₂·z⁻²
  • 可配置 N 段 EQ（典型 5 段）
  • Q31 定点运算（RBJ 系数 → 定点转换）
  • 音量平滑：一阶 IIR，Q15 系数 VOLUME_SMOOTH_ALPHA_Q15 = 33
