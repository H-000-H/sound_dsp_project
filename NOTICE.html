💀 13 个坑 · 速查

1. WDT 修改后必须 fullclean — 改 sdkconfig 禁用 WDT 后增量 build 仍编译旧代码。
✅ idf.py fullclean（或删 build/ 目录）后重编。reconfigure 不够。
2. LVGL 9.5 group 吞噬 PREV/NEXT — KEYPAD indev 下 PREV/NEXT 被 group 焦点管理消耗。✅ GPIO 轮询绕过：lv_timer 定时读 get_pressed_gpio() 独立处理。
3. WDT 条件编译分多层 — CONFIG_ESP_TASK_WDT_EN（主开关）、_INIT（启动初始化）、BOOTLOADER_WDT。须全部设为 not set。
4. lv_lottie.h 强制要求 ThorVG — lv_conf_internal.h 默认 LV_USE_LOTTIE=1 但 Kconfig 注释掉。✅ 顶层 CMakeLists.txt 加 -DLV_USE_LOTTIE=0，源码用 #if LV_USE_LOTTIE 包裹。
5. PSRAM Quad vs Octal 模式不匹配 — 硬件 Octal 但默认 Quad 导致 PSRAM chip not connected 反复重启。✅ sdkconfig 设 SPIRAM_MODE_OCT=y。
6. 显示缓冲区放 DRAM 导致内存爆 — 56KB 缓冲区默认 MALLOC_CAP_DMA 占满 DRAM。✅ 非 LVGL 缓冲区（串口 ring buffer 等）用 MALLOC_CAP_SPIRAM。但 LVGL 绘制缓冲区例外（见第 14 坑）。
7. friend + static 重复声明 — music_app.hpp 同时有 friend 和 static 声明同一函数。✅ 移除 static。
8. 全局变量重复定义 — music_impl.cpp 重复定义 g_song_list（已存在于 song_list.cpp）。✅ 删重复。
9. 串口换行后颜色丢失 — buf[0]=='I' && buf[1]==' ' 条件太严格。✅ 改用 buf[0]=='I' 单字符判断。
10. WiFi 断连后无法重连 — 只调 disconnect() 不够。✅ 超时后 wifi->stop() 完全停止驱动，重连前 start()。
11. LVGL 堆 32KB → 爆死锁 — draw layer buffer 耗尽。✅ LV_MEM_SIZE 增大到 128KB。
12. 高帧率 + WDT = 更容易触发 — 帧率越高 timer 回调越频繁，阻塞时 WDT 更快超时。✅ 本项目禁用 WDT（非工业场景不需要）。
13. ST7789 7 引脚无 CS — 必须 SPI Mode 3 — 该模组没有独立 CS 引脚（spics_io_num = -1），DC 代替命令/数据区分。SPI 必须设为 Mode 3（CPOL=1, CPHA=1），否则 ST7789 无法正确采样 SCLK 边沿导致花屏或不显示。✅ spi_bus.cpp interface_cfg.mode = 3，同时设 flags = SPI_DEVICE_HALFDUPLEX（半双工，7 线无 MISO）。
14. LVGL 帧缓冲放 PSRAM → WiFi 重连时花屏/卡死 — ESP32-S3 PSRAM 通过 CPU cache 访问，WiFi bcn_timeout 触发重连 → flash/NVS 操作 → cache 禁用 → PSRAM 数据损坏。✅ lvgl_main.cpp 新增 lvgl_alloc_buf()，LVGL 绘制缓冲区优先从内部 RAM（MALLOC_CAP_INTERNAL | MALLOC_CAP_DMA）分配，仅在 USE_LVGL_PRAM=1 时用 PSRAM。

🔑 LVGL 9.5 group 吞噬详解

核心问题：LV_INDEV_TYPE_KEYPAD 下，PREV/NEXT 被 group 内部焦点导航消耗，不会发送 LV_EVENT_KEY。ENTER/ESC 正常（group 不处理它们）。
当前影响：音乐播放器、串口调试、歌曲列表 → PREV/NEXT 无法切换焦点区。卡片菜单正常（GPIO 轮询独立驱动）。
方案一（推荐）：GPIO 轮询 — lv_timer 定时读 Button::get_pressed_gpio()，独立于 group。
方案二：lv_group_set_editing(true) — ENTER 进入编辑模式后 PREV/NEXT 由对象自身处理。
